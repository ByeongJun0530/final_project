# 📌 데이터 전처리 파이프라인

## STEP 1. 제품별 데이터 분리
제품 특성에 맞게 데이터를 분리하여 모델을 학습한다.

- `A_31`
- `T_31`, `O_31`

단, 제품 구분을 위해 `PRODUCT_CODE`는 유지한다.

---

## STEP 2. 누수(Leakage) 컬럼 제거
모델 학습 시 타깃 정보가 포함되지 않도록 누수 컬럼을 제거한다.

- 타깃 컬럼 `Y_Quality` 제거
- `PRODUCT_ID`, `TIMESTAMP`는 모델 입력 피처에서 제외
- 단, `TIMESTAMP`는 시간 기반 검증을 위해 별도로 보관

---

## STEP 3. 위장 결측값 정규화
실제 결측값을 의미하는 placeholder 값을 모두 `NaN`으로 통일한다.

**변환 대상 예시**
- `999`
- `9999`
- `-999`
- `-9999`

> 이후 결측 비율·분산 계산이 위장 결측을 실제 결측으로 반영할 수 있도록, 다른 결측/분산 처리보다 먼저 수행한다.

---

## STEP 4. 완전 결측 컬럼 제거
모든 값이 `NaN`인 컬럼을 제거한다.

- 정보량이 전혀 없는 컬럼이므로 제거

---

## STEP 5. 결측 비율 60% 초과 컬럼 제거
결측 비율이 기준치(기본 60%)를 초과하는 컬럼을 제거한다.

- `threshold` 파라미터로 기준 비율 조정 가능
- 정보량이 지나치게 부족한 컬럼을 정리하여 피처 수를 축소

---

## STEP 6. 상수 · 제로분산 컬럼 제거
정보량이 없는 센서 컬럼을 제거한다.

**제거 기준**
- `nunique() ≤ 1`
- `std() = 0`

---

## STEP 7. 저분산 컬럼 제거
값이 대부분 비슷하게 유지되어 예측에 기여도가 낮은 컬럼을 제거한다.

- 컬럼별 min-max 정규화 후 분산 계산 (스케일 차이로 인한 왜곡 방지)
- 정규화된 분산이 기준치(기본 0.01) 미만이면 제거

---

## STEP 8. 상관계수 높은 컬럼 제거
서로 강하게 상관된 컬럼을 제거하여 다중공선성과 중복 정보를 줄인다.

- 수치형 컬럼 간 상관계수(절대값) 계산
- 기준치(기본 0.95)를 초과하는 컬럼 쌍 중 한쪽을 제거

---

## STEP 9. 범주형 변수 인코딩
모델 학습을 위해 범주형 변수인 `LINE`을 인코딩한다.

- `LINE` 변수 인코딩
- 머신러닝 모델 입력 형태로 변환

---

## STEP 10. 결측값 처리 정책
트리 기반 모델의 특성을 고려하여 남은 결측값을 별도로 대체하지 않는다.

- 결측값(`NaN`) 그대로 유지
- 불필요한 평균·중앙값 대체(Imputation) 수행하지 않음

---

## 모델링 설계 (참고)
타깃 기반 피처 선택(타깃 상관 필터 등)은 리키지 방지를 위해 전처리 단계가 아닌
모델링 단계에서 Cross Validation 내부에 포함하여 수행한다.

### 타깃 설계
- `Y_Quality`를 예측하는 회귀 모델 학습
- 예측 결과를 임계값(Threshold)을 기준으로 품질 등급으로 분류

### 클래스 불균형 처리
- 클래스 가중치(Class Weight)를 적용하여 불균형 완화

### 검증 방법
- 데이터 누수를 방지하기 위해 **Stratified K-Fold Cross Validation** 사용
- 각 Fold의 클래스 비율을 유지하며 모델 성능 평가
